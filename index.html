The Same Sorting Problem, Three Geometries

The manuscript keeps the input values, original indices, rank words, and sorted index order separate. If \(v_s=(1,2,\ldots,n)\), the permutohedron \(\mathcal P_n=\operatorname{conv}\{\pi(v_s):\pi\in S_n\}\) becomes the common stage for local motion, global comparison cuts, and continuous relaxation.

1

Decision Tree

A comparison algorithm must distinguish all \(n!\) order types. A binary tree of height \(h\) has at most \(2^h\) leaves, so \(2^h\ge n!\) and \(h\ge\log_2(n!)=\Theta(n\log n)\).

2

Permutohedron

Each vertex is a rank word. Adjacent transpositions form edges of the 1-skeleton, so local algorithms walk through neighboring vertices and pay for inversions one at a time.

The Gradient-Flow Benchmark

The continuous model is deliberately a relaxation. It does not say that MergeSort or QuickSort follows a Euclidean trajectory. It gives a precise contraction law for rank displacement, then compares that macroscopic clock with the information supplied by discrete comparisons.

$$V(x)=\frac{1}{2}\|x-v_s\|^2$$
$$\dot{x}(t)=-\nabla V(x(t))=v_s-x(t)$$
$$\|x(t)-v_s\|^2=\|x(0)-v_s\|^2e^{-2t}$$
$$\dot{x}(t)=\Pi_{T_{K(C)}(x(t))}\bigl(-\nabla V(x(t))\bigr)$$
The last line is the projected version after comparison outcomes define a feasible convex region \(K(C)\). Projection encodes the active comparison constraints while the potential still measures remaining displacement from the sorted rank word.
